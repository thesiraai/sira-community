# Contributing to SIRA Community

SIRA Community is a fork of [Discourse](https://github.com/discourse/discourse). We welcome contributions that improve SIRA Community's integration with the SIRA AI ecosystem.

## Important note for Developers

### Contributing to SIRA Community

For contributions specific to SIRA Community (SIRA integrations, custom features, etc.):

1. Fork the SIRA Community repository
2. Create a feature branch
3. Make your changes
4. Submit a pull request with a clear description

### Contributing to Discourse (Upstream)

If your contribution would benefit the broader Discourse community, we encourage you to contribute upstream to the [official Discourse repository](https://github.com/discourse/discourse).

Anyone wishing to contribute to Discourse **must read & sign the [Contributor License Agreement](http://www.discourse.org/cla)**. The Discourse team is legally prevented from accepting any pull requests from users who have not signed the CLA first.

For more information on contributing to Discourse:

- [Discourse Development Contribution Guidelines](https://meta.discourse.org/t/discourse-development-contribution-guidelines/3823)
- [How to set up your development environment](https://meta.discourse.org/t/beginners-guide-to-install-discourse-for-development-using-docker/102009)
- [First-time project suggestions](https://meta.discourse.org/t/so-you-want-to-help-out-with-discourse/3823)
- [Code conventions and style guide](https://meta.discourse.org/t/discourse-development-contribution-guidelines/3823)

## Ways to Contribute

There are many ways to contribute to SIRA Community:

### Code Contributions
- Bug fixes
- New features for SIRA integration
- Performance improvements
- Documentation updates

### Non-Code Contributions
- Reporting bugs
- Requesting features
- Improving documentation
- Testing and quality assurance
- Translation (if applicable)
